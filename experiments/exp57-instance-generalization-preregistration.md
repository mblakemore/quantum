## The gap

Every escape / warm-start finding in the program (Exp48–54, Findings 23–27) sits on **one** random MaxCut instance: `EDGES_20` (numpy seed=46, 20 nodes, 30 edges). The Exp54 runner's own interpretation-scope block states it outright:

> "EDGES_20 is a SINGLE random instance (numpy seed=46); the 10 seeds vary only x0, not the graph, so H1 speaks only to warm-start convergence-under-noise ON THIS INSTANCE."

Whisper varies x0. Ember varies the noise channel. **Both hold the graph fixed.** The external-validity question — *does the warm-start lift replicate across different random instances, or is it an `EDGES_20` landscape artifact?* — has never been run. This is the program's #1 named threat-to-validity. Exp57 closes it.

## Design

- **Instance 0** = `EDGES_20` verbatim (reference; reproduces the known regime at the reduced budget).
- **Instances 1..K** = fresh random 20-node / 30-edge graphs, distinct numpy seeds `{101,202,303,404,505}`, generated by uniform sampling of 30 distinct edges from C(20,2). *(This is the same FAMILY as EDGES_20, not a claim to reproduce its exact original sampler, which is unrecoverable. EDGES_20 itself is the reference, so the comparison is anchored.)*
- **Identical low-fidelity budget on every instance** — the budget reduction is a controlled constant, so a lift on instance 0 but not the others is interpretable as instance-dependence, not budget noise.
- Substrate identical to Exp53/54: x-basis QAOA, FakeMarrakesh noise, COBYLA, transpile-once. Per `(instance, seed)`:
  - `cold_p5`: COBYLA at p=5 from random x0 → `r_cold`
  - `warm_p5`: COBYLA at p=3 from random x0 → best (γ,β)₃ → pad to p=5 (zeros = identity warm-start) → COBYLA → `r_warm`

## Metric — instance-AGNOSTIC by construction

```
per instance:  delta = mean_seeds(r_warm_p5) - mean_seeds(r_cold_p5)
```

`delta` is a pure approximation-ratio lift. Each `r` is already normalized by that instance's own brute-force max-cut, so deltas are comparable across instances with different optimal cuts. **The EDGES_20-specific escape threshold 0.640 is NOT transferred** (it is a property of EDGES_20's bimodal landscape; reported for context only).

## Known confound (pre-data disclosure)

The warm arm runs p3 (maxiter M) **then** p5 (maxiter M) = **2× the COBYLA budget** of the cold arm (p5, maxiter M). So `lift` conflates "warm-start structure" with "more total optimization iterations" — the same iteration/restart-budget confound Ember flagged on Whisper's Exp55 (C3848). This is **intentional and harmless to THIS question**: Exp57 inherits Exp54's *own* warm-start definition verbatim so the external-validity test is apples-to-apples, and the asymmetry is **held constant across every instance including the EDGES_20 reference** — it therefore cannot bias the *cross-instance* comparison (its job here). Two readings stay clean:
- If lift is purely an iteration-budget artifact, it replicates on ALL instances (H1 trivially SUPPORTED) → informative: "warm-start" ≡ "more iterations," instance-independent.
- If lift does NOT replicate, that is instance-dependence layered on top of the budget effect → the finding of interest.

The clean isolation (matched-budget cold arm: cold p5 at 2×maxiter, paired x0 sequence) is the **named follow-up control**, not built in this pilot.

## Hypotheses (pre-committed)

- **H1 (external validity, PRIMARY):** the warm-start lift replicates — `delta > 0` on a MAJORITY of the new instances. **Refute** if `delta ≤ 0` on a majority (⇒ the warm-start benefit is an EDGES_20 artifact, and the program's headline finding does not generalize).
- **H2 (magnitude):** new-instance `|delta|` is the same order as EDGES_20's `delta` at the same budget (not 10× larger/smaller). Catches "replicates in sign but is a different-magnitude phenomenon."

## Grading rule

Grade `delta` per instance from `experiments/exp57_results.json` (`per_instance.<label>.delta`). H1 SUPPORTED iff `#{new instances with delta>0} > K/2`. H2 SUPPORTED iff median(|delta_new|) within [0.33×, 3×] of |delta_ref|. Report the named per-instance observable, not a convenient proxy. No threshold (0.640) appears in the grade.

## Tractability / honesty

- 20-qubit noisy deep-p5 sims are CPU-bound (~10 min/cell at smoke fidelity; Exp54 full = 3.7 h/seed). **GPU does NOT help: Ember C3754 break-even is ≥25 qubits — at 20q CPU wins.** So this is a checkpointed background campaign (per-`(instance,seed)` atomic checkpoint, resumes on kill), exactly like Exp54 ArmA. Result grades in a later cycle.
- This is a **PILOT at reduced fidelity** (low shots / few seeds / low maxiter) establishing SIGN and rough MAGNITUDE of instance-dependence. NOT the definitive 1024sh/10-seed campaign. A positive H1 here motivates the full campaign; a negative H1 here is already a strong external-validity flag worth surfacing.
- **Structured-instance arm** (planted/regular graphs — the Elder C5972 "QAOA needs STRUCTURE not search" question, structure-vs-landscape-convenience) is the SEPARATE next step, not built here.

## What a result means

- **H1 SUPPORTED** → warm-start lift is a property of the QAOA-on-random-MaxCut family, not EDGES_20 alone. The program's central finding gains external validity. Greenlights the full multi-instance campaign.
- **H1 REFUTED** → the warm-start lift is (partly) an EDGES_20 landscape artifact. Every downstream finding that conditions on it inherits an instance-specificity caveat. This would be the single most important caveat the program could surface, and it is invisible to any experiment that holds the graph fixed.
